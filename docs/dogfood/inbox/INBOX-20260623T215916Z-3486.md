## Suspected fix
Invariants validated/derived from the on2it-pr-council reference workflow (see
docs/design/loop-architecture.md §3, §6):
- Severity veto: a substantiated high-confidence BLOCK with concrete blocking_issues
  forces NO_CONSENSUS (escalate to human), not a simple-majority override.
- Execution-evidence-required quorum: no APPROVE/winner without a present, passing
  test-gate verdict; a dead reviewer is not an abstention that lowers the threshold.
- Compute tally/consensus/elected in code; the writer agent produces narrative only.
- Reviewer output schema: { vote, confidence, rationale, blocking_issues[],
  nonblocking_notes[], checks_run[] }; deliberation adds { changed_from_round1,
  response_to_peers, final_rationale }.
- Deliberation is one simultaneous round against peers' pre-deliberation positions;
  if r1->r2 flips the outcome (esp. BLOCK->APPROVE), prefer NO_CONSENSUS and surface it.
- All lenses pinned to one commit SHA, never a moving branch ref.
